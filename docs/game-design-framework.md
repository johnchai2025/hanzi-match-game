## 一、背景：这是什么，为谁而做

### 项目定位

**「汉字对对碰」是一款为 6-9 岁小学低年级儿童设计的汉字组词练习游戏**，以轻松的消除玩法替代枯燥的汉字抄写和记忆任务。

核心使用场景：**家庭辅助教学**。父母陪孩子玩，或孩子独立操作（有提示系统兜底）。家长可通过"自定义词库"功能上传语文课本生词，实现课内外学习无缝衔接。

### 目标用户

| 角色 | 年龄 | 使用方式 |
|---|---|---|
| 主要用户：孩子 | 6-9 岁（小学 1-3 年级） | 点击格子，完成消除 |
| 辅助用户：父母 | 成年人 | 设置 IP/场景偏好、上传词库、陪伴游玩 |

**用户偏好数据（`ProfileSetupModal`）**：
- 孩子名字（个性化称呼）
- 喜欢的 IP：小猪佩奇、奥特曼、皮卡丘、叶罗丽、小马宝莉、汪汪队、海底小纵队、光之美少女
- 喜欢的场景：森林、学校、家里、太空、海边、城市、糖果王国

### 作者

`v1.5 · 哈维yao`，个人/家庭项目，非商业产品。

## 二、教育目标：练什么，怎么练

### 核心练习目标

> **通过"找词配对"的游戏行为，强化双字词语的识别、记忆和联想能力**

具体分解：

| 能力维度 | 游戏中的对应行为 |
|---|---|
| 词语视觉识别 | 在 36 个混排汉字格中，找出能组成同一词语的两个字 |
| 组词联想 | 看到"高"时，需联想出配对的"山"在哪里 |
| 主题词汇积累 | 每关围绕一个生活主题（天地自然、动物、职业…），形成系统化词汇图谱 |
| 图文联想记忆（设计意图） | AI 词卡将词语与插图绑定，强化图像记忆路径 |

### 学习路径设计

```
玩游戏 → 消除词语对 → 触发 AI 词卡生成 → 词卡存入"词语本"
                                              ↓
                                   游戏结束后可回顾词卡
                                   （图 + 词 + 例句/场景画面）
```

### 练习的边界（不练什么）

- **不练字形书写**：游戏只涉及认读，不要求书写笔画
- **不练拼音**：纯汉字界面，无拼音辅助（面向已认识基础字的阶段）
- **不练单字发音**：暂无 TTS 朗读功能（规划中但未实现）
- **不练造句**：词语在具体语境中的应用不在当前范围内

## 三、游戏玩法框架

### 核心循环

```
选关 → 棋盘初始化（36格，每格一个汉字）
     → 玩家扫视棋盘，找到能组词的两个字
     → 点击第一个字（高亮选中）
     → 点击第二个字
         ├── 两字来自同一词对？→ 消除（动画 + 词卡生成触发）
         └── 不是同一词对？→ 选中转移到新点击的字
     → 消除18对 → 通关（过关弹窗 + 解锁下一关）
```

### 棋盘规则

| 参数 | 值 | 说明 |
|---|---|---|
| 棋盘尺寸 | 6×6 | 固定36格，全程不变 |
| 每关词对数 | 18对 | 不足18对则取全部；超过18对随机抽取18对 |
| 棋盘填充 | 每对词产生2个单字格子 | 18对×2=36格，恰好填满 |
| 路径限制 | 无 | 无需连线，纯词语配对 |
| 顺序要求 | 无 | 先点哪个字都可以 |

### 辅助机制

| 机制 | 触发方式 | 作用 |
|---|---|---|
| 提示 | 点击"提示"按钮 | 高亮随机一对可消除的格子（2秒后消失） |
| 里程碑激励 | 消除到一半时自动触发 | 显示"已经消了一半啦！继续加油 ⚡"（仅一次） |
| 死局检测 | 每次消除后自动检测 | 若剩余格子无任何可配对词对，弹出死局弹窗 |
| 重新打乱 | 死局弹窗中点击 | 保留消除进度，只重新排列剩余格子位置 |
| 重新开始 | 死局弹窗 / 游戏中菜单 | 重置本关全部进度 |

### 消除条件的设计意图

棋盘上每格显示**单个汉字**（如"高"、"山"），玩家需要认出这两个字能组成词语"高山"。这是游戏与其他消消乐的本质区别：

- **不是视觉匹配**（不是找相同图案）
- **是词语认知**（需要知道"高+山=高山"才能消除）

这个设计决策决定了一切：格子必须显示单字，而不是整个词语。显示完整词语则退化为普通的连连看，失去教育价值。

## 四、关卡结构

### 三级五关体系

| 级别 | 名称 | 关卡主题 | 词汇特点 |
|---|---|---|---|
| Grade 1 | 第1级·启蒙 | 天地自然、动物朋友、身体部位、我的家、花草植物 | 具象事物，字形简单（天空、月亮、手指、爸爸） |
| Grade 2 | 第2级·进阶 | 食物美味、交通出行、学校生活、四季变化、职业人物 | 社会性概念，词汇略复杂（馄饨、骆驼、腊梅） |
| Grade 3 | 第3级·挑战 | 自然现象、情绪感受、品德修养、地理知识、时间概念 | 抽象词汇为主（霜降、嫉妒、谦虚、黎明、永远） |

### 解锁机制

- 初始仅开放 `1-1`（天地自然）
- 每完成一关，自动解锁下一关
- 跨级别无特殊解锁要求（完成 1-5 自动解锁 2-1）
- 已完成的关卡可重复游玩（不重置解锁进度）

### 自定义关卡

- 父母可上传 `.txt` 文件或粘贴词语（每行一个双字词）
- 解析规则：过滤非汉字字符，只保留恰好2个汉字的行，最低18个词
- 自定义关卡与主线关卡完全独立，不影响解锁进度
- 存储在 localStorage，无云端同步

## 八、评估标准：什么叫"开发成功"

基于以上设计框架，评估任何功能迭代的依据如下：

### 6.1 教育目标达成

| 评估维度 | 标准 |
|---|---|
| 棋盘格子显示 | **必须**是单个汉字，而非完整词语 |
| 消除判断 | 两个字来自同一词对即可消除，无顺序要求 |
| 难度递进 | Grade 1 → Grade 2 → Grade 3 词汇难度明显提升 |
| 主题聚焦 | 每关词语围绕同一主题（不能随机混搭无关主题） |

### 6.2 玩法体验

| 评估维度 | 标准 |
|---|---|
| 棋盘可读性 | 6×6格子在手机屏幕上汉字清晰可辨（字号不小于18px） |
| 操作流畅度 | 点击到消除动画流畅，无明显卡顿 |
| 提示有效性 | 点击提示后，高亮的格子确实是可以消除的一对 |
| 死局处理 | 死局时弹窗出现及时，"重新打乱"后棋盘有新的可消除对 |
| 进度不丢失 | 刷新页面后，已解锁关卡和已完成关卡状态正确恢复 |

### 6.3 AI 词卡功能（待完成）

| 评估维度 | 标准 |
|---|---|
| 触发时机 | 每次消除后触发词卡生成（异步，不阻塞游戏） |
| 图片内容 | 图片与词语主题相关，适合6-9岁儿童，无不适内容 |
| 个性化 | 图片风格体现用户设置的 IP 和场景偏好 |
| 错误降级 | API 失败时词卡以文字版显示，游戏不中断 |

### 6.4 不在范围内的功能（明确边界）

以下需求若提出，需先讨论是否符合设计初衷再决定是否实现：

- ❌ 棋盘格子显示完整词语（退化为普通连连看，丧失教育价值）
- ❌ 任意两个汉字都能消除（失去词语认知练习的目的）
- ❌ 无限关卡自动生成（需保证词汇质量，不能用随机组合替代人工编排）
- ⚠️ 识别跨词对的合法词（如"山水"、"大火"）：需引入词典，是中期目标，不是当前范围

## 七、技术架构约束

| 约束 | 说明 |
|---|---|
| 框架 | Next.js 15 (App Router)，TypeScript，React |
| 存储 | localStorage（无后端数据库，无用户账户） |
| 棋盘大小 | 固定 6×6，与18对词的数量绑定，不可随意改变 |
| 词对格式 | `[string, string]`，两个单字合并为双字词 |
| 关卡数据 | `public/levels.json`，每关必须有18对词语（当前校验逻辑限制） |
| AI 接入 | Gemini API（Imagen 3 生图 + Flash 生故事），需环境变量 `GEMINI_API_KEY` |
