# 汉字对对碰 · 产品规格文档

> 面向 Claude Code 的完整开发规格  
> 目标用户：小学低年级（6-9岁）  
> 技术栈：React + TypeScript  
> 模式：自由练习（无时间/步数限制）  
> 部署方式：打包静态文件，部署到服务器  
> 字库管理：**字库外置为独立 JSON 文件**，在服务器上直接编辑即可更新，无需重新编译
> 自定义字库：**用户可上传 txt 文件**，作为自定义关卡永久保存在本地 localStorage，随时重玩

---

## 一、产品概述

### 核心玩法

- 棋盘为 **6×6 方格**，共 36 个格子
- 每局游戏从当前关卡字库中随机抽取 **18 组词对**，将 36 个汉字随机打乱铺满棋盘
- 每个词对由两个字组成，必须按**前字在前、后字在后**的顺序组成合法词语（如「苹」+「果」=「苹果」✓，「果」+「苹」✗）
- 玩家点击两个格子，若这两个字能按顺序组成词语，则**消除**这两个格子
- 全部消除完毕，过关

### 胜负条件

- **胜利**：所有 36 个格子全部消除
- **失败**：无（自由模式，无限重试）
- **提示**：提供「提示」按钮，高亮一个可消除的词对

## 三、交互逻辑

### 点击行为

1. 玩家点击第一个格子 → 该格子高亮（选中态）
2. 玩家点击第二个格子：
   - **若两字可组词**（第一字在前）→ 消除动画 → 两格消失
   - **若顺序反了**（第二字在前才合法）→ 提示「试试换个顺序」，不消除，清除选中
   - **若无法组词** → 取消选中，重新选择
3. 消除后，**空格保留为空**（不下落补位，保持原位置为空）

### 选中态规则

- 同一格子点击两次 → 取消选中
- 选中第一个字后，点击第三个格子 → 第一个格子取消，第三个格子成为选中

### 提示功能

- 点击「提示」→ 随机高亮棋盘上一个合法词对的两个字（绿色边框闪烁动画，持续 2 秒）
- 提示不消耗次数（自由模式无限制）

### 过关动画

- 最后一对消除后，显示「太棒了！🎉」全屏庆祝动画
- 自动显示「下一关」按钮（若当前非最后一关）

## 四、关卡结构

### 分级体系

| 级别 | 名称 | 关卡数 | 词语特点 |
|------|------|--------|----------|
| 第1级 | 启蒙 | 5关 | 一年级常用字，2字词，结构简单 |
| 第2级 | 进阶 | 5关 | 二年级扩展字，稍复杂词汇 |
| 第3级 | 挑战 | 5关 | 三年级词汇，抽象概念词 |

- 总计：**3级 × 5关 = 15关**
- 每关：**18组词对**（36个字）
- **字库设计原则：每关36个汉字全部唯一，无任何重复**
- 关卡解锁：完成当前关解锁下一关；完成第5关解锁下一级第1关

### TypeScript 类型定义

```typescript
type WordPair = [string, string]; // [前字, 后字]

interface LevelData {
  id: string;
  grade: 1 | 2 | 3;
  level: number;
  title: string;
  pairs: WordPair[];
}

interface Cell {
  id: string;                   // "cell-r-c"
  char: string;                 // 显示汉字
  pairId: number;               // 所属词对索引 0~17
  role: "first" | "second";    // 前字 or 后字
  isEmpty: boolean;
  isSelected: boolean;
  isHinted: boolean;
}

interface GameState {
  currentLevel: LevelData;
  cells: Cell[][];              // 6×6
  selectedCell: { row: number; col: number } | null;
  eliminatedCount: number;      // 已消除词对数 0~18
  isComplete: boolean;
  feedbackMessage: string | null;
}

interface SaveData {
  customLevels: CustomLevel[];  // 用户上传的自定义关卡，存 localStorage
  unlockedLevels: string[];     // 初始为 ["1-1"]
  completedLevels: string[];
}
```

### 棋盘初始化算法

```typescript
function initBoard(level: LevelData): Cell[][] {
  // 展开18个词对为36个字
  const allCells = level.pairs.flatMap((pair, pairId) => [
    { char: pair[0], pairId, role: "first" as const },
    { char: pair[1], pairId, role: "second" as const },
  ]);

  // Fisher-Yates 随机打乱
  for (let i = allCells.length - 1; i > 0; i--) {
    const j = Math.floor(Math.random() * (i + 1));
    [allCells[i], allCells[j]] = [allCells[j], allCells[i]];
  }

  // 填充6×6棋盘
  return Array.from({ length: 6 }, (_, r) =>
    Array.from({ length: 6 }, (_, c) => ({
      id: `cell-${r}-${c}`,
      ...allCells[r * 6 + c],
      isEmpty: false,
      isSelected: false,
      isHinted: false,
    }))
  );
}
```

### 消除判断逻辑

```typescript
function canEliminate(a: Cell, b: Cell): boolean {
  // 同一词对，且 a 是前字，b 是后字
  return a.pairId === b.pairId && a.role === "first" && b.role === "second";
}

function isReversedOrder(a: Cell, b: Cell): boolean {
  // 同一词对，但顺序反了
  return a.pairId === b.pairId && a.role === "second" && b.role === "first";
}
```

### txt 文件格式规范

每行一个完整词语（2个汉字），游戏自动拆成前字 + 后字：

```
苹果
香蕉
西瓜
天空
月亮
...（至少18行，推荐18~36行）
```

**格式规则：**
- 每行恰好 2 个汉字，多余字符（空格、标点）自动忽略
- 空行自动跳过
- 文件编码：UTF-8（Windows 用记事本保存时选「UTF-8」）
- 最少需要 18 个有效词语（满足一局游戏所需）
- 超过 18 个词语时，每次开始游戏从中**随机抽取 18 个**，增加复玩性
- 不足 18 个时，上传时提示错误，不保存

### 解析与校验逻辑

```typescript
interface ParseResult {
  pairs: WordPair[];       // 解析成功的词对
  errors: string[];        // 错误信息列表
  warnings: string[];      // 警告信息列表（不阻止保存）
}

function parseTxtFile(content: string): ParseResult {
  const lines = content.split('\n');
  const pairs: WordPair[] = [];
  const errors: string[] = [];
  const warnings: string[] = [];

  lines.forEach((raw, i) => {
    const line = raw.trim().replace(/[^\u4e00-\u9fff]/g, ''); // 只保留汉字
    if (line.length === 0) return;           // 跳过空行
    if (line.length !== 2) {
      warnings.push(`第${i + 1}行「${raw.trim()}」不是2个汉字，已跳过`);
      return;
    }
    pairs.push([line[0], line[1]]);
  });

  if (pairs.length < 18) {
    errors.push(`有效词语不足18个（当前：${pairs.length}个），请补充后重新上传`);
  }

  return { pairs, errors, warnings };
}
```

### 游戏时的抽词逻辑

自定义关卡词对数可能超过 18 个，每次开始游戏时随机抽取：

```typescript
function pickPairsForGame(level: CustomLevel): WordPair[] {
  if (level.pairs.length <= 18) return [...level.pairs];
  // Fisher-Yates 随机抽18个
  const shuffled = [...level.pairs];
  for (let i = shuffled.length - 1; i > 0; i--) {
    const j = Math.floor(Math.random() * (i + 1));
    [shuffled[i], shuffled[j]] = [shuffled[j], shuffled[i]];
  }
  return shuffled.slice(0, 18);
}
```

> 注意：自定义关卡不保证每关36字唯一（用户提供的词可能含重复字），游戏逻辑本身不受影响，但提示文案可调整为「同一个字可能属于不同词语，请注意搭配」。

## 十、字库外置与自定义编辑

### 设计目标

字库不硬编码在源码中，而是作为独立的 `levels.json` 文件存放在服务器上。你只需用文本编辑器修改这个文件并上传覆盖，玩家刷新页面后即自动套用新内容，**无需重新编译或修改任何代码**。

### 文件位置

```
dist/（打包产物，部署到服务器的目录）
├── index.html
├── assets/
│   └── ...（JS/CSS，不需要动）
└── levels.json        ← 你只需要编辑和替换这一个文件
```

构建配置要求：`levels.json` 放在项目的 `public/` 目录，Vite 打包时会原样复制到 `dist/`，不做任何处理。

### levels.json 完整格式规范

```json
{
  "version": "1.0",
  "levels": [
    {
      "id": "1-1",
      "grade": 1,
      "level": 1,
      "title": "天地自然",
      "pairs": [
        ["天", "空"],
        ["月", "亮"],
        ["太", "阳"]
      ]
    }
  ]
}
```

**字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `version` | string | 格式版本号，目前固定为 `"1.0"` |
| `levels` | array | 所有关卡数组，顺序即为游戏中显示顺序 |
| `id` | string | 关卡唯一标识，格式为 `"级别-关卡"`，如 `"1-1"` |
| `grade` | number | 级别，目前为 1 / 2 / 3 |
| `level` | number | 该级别内的关卡序号，从 1 开始 |
| `title` | string | 关卡标题，显示在游戏界面 |
| `pairs` | array | 词对数组，每项为 `[前字, 后字]`，**恰好 18 个** |

### 编辑规则（重要）

1. **每关恰好 18 个词对**，展开后 36 个汉字，**每个汉字在本关内不得重复出现**
2. 词对顺序为 `[前字, 后字]`，只有「前字 + 后字」才是合法词语
3. 修改 `id` 字段会导致已保存的通关进度失效（localStorage 以 id 为键），**不建议修改已有关卡的 id**
4. 可以自由增减关卡总数，游戏会动态渲染关卡选择界面
5. 新增关卡时，`grade` 和 `level` 仅影响分组显示，不影响游戏逻辑

### 游戏启动加载逻辑

游戏启动时通过 `fetch` 加载字库，加载期间显示 loading 状态；加载失败时显示错误提示（而非白屏）：

```typescript
// src/hooks/useLevels.ts
export function useLevels() {
  const [levels, setLevels] = useState<LevelData[]>([]);
  const [error, setError] = useState<string | null>(null);
  const [loading, setLoading] = useState(true);

  useEffect(() => {
    fetch('/levels.json')
      .then(res => {
        if (!res.ok) throw new Error(`HTTP ${res.status}`);
        return res.json();
      })
      .then(data => {
        // 基础校验：每关必须有18个词对
        const valid = data.levels.every(
          (l: LevelData) => Array.isArray(l.pairs) && l.pairs.length === 18
        );
        if (!valid) throw new Error('字库格式错误：存在关卡词对数量不等于18');
        setLevels(data.levels);
      })
      .catch(err => setError(err.message))
      .finally(() => setLoading(false));
  }, []);

  return { levels, loading, error };
}
```

### 本地开发时的调试方法

虽然最终部署为静态文件，本地开发时（`npm run dev`）`levels.json` 同样放在 `public/` 目录，Vite dev server 会直接 serve，无需额外配置。修改 `levels.json` 后**刷新浏览器**即可看到效果。

### 字库校验工具（可选，供 Claude Code 实现）

在项目中提供一个独立的校验脚本 `scripts/validate-levels.js`，方便你在上传前本地检查：

```bash
node scripts/validate-levels.js public/levels.json
```

校验内容：
- 每关词对数量是否恰好为 18
- 每关 36 个汉字是否有重复
- JSON 格式是否合法
- id 是否唯一
